## 2. 开场话术

> "各位好，我展示的项目是 **CiteWise**——一个以用户私有知识库为核心的智能研究助手。
>
> 先说一个数据：学术研究中，文献梳理和论文写作占研究者 60% 以上的时间。但现有工具要么只管文献（Zotero），要么只做问答（Elicit），要么通用生成但不靠谱（ChatGPT 容易幻觉）。
>
> CiteWise 要解决的核心问题是：**如何让 AI 在学术写作场景下既高效又可信？**
>
> 我的答案是三个关键词：**可定制的结构化总结、强制溯源防幻觉、个性化记忆复用**。"

---

## 3. 核心差异化展示

### 3.1 可定制的结构化总结

> "现有工具（Elicit、SciSpace）提供固定的提取字段。但不同学科关注点完全不同——交通领域关注'空间范围'和'数据来源'，CV 领域关注'骨干网络'和'mAP'。
>
> CiteWise 让用户自定义字段，Agent 批量从每篇文献中提取，生成对比表格和可视化图表。
>
> **展示**：定义 [方法, 数据集, 核心指标, 创新点] → 自动生成对比表格 + 柱状图"

### 3.2 强制溯源防幻觉

> "这是 CiteWise 的信任基石。三层防护：
> 1. System Prompt 层面要求每个观点必须引用
> 2. 后处理校验引用是否在知识库中
> 3. 无依据内容自动标记 [未验证]
>
> **展示**：生成一段文献综述，每个观点后都有 [作者, 年份]，点击可跳转原文"

### 3.3 个性化记忆复用

> "ChatGPT 每次对话从零开始。CiteWise 记住你的研究领域、字段偏好、写作风格。
>
> 新建项目时，系统主动推荐：'检测到您之前使用过 [方法, 数据集, 指标] 模板，是否沿用？'
>
> **展示**：创建新项目 → 系统自动推荐历史模板和框架"

## 4. 技术架构讲解

> "技术架构分四层，核心是五大能力的系统化设计：

### 画图讲解

```
用户层：对话 + 上传 + 表格查看 + 导出
    ↓
Agent 层：ReAct 规划器 + 10 个专用工具
    ↓
能力支撑层：Prompt / RAG / 上下文 / 记忆 / 多模态
    ↓
数据层：向量库 + 图表库 + 用户画像 + 项目记录
```

> 重点讲两个：

### RAG

> "学术论文不能粗暴按字数切。我们用三级切片（论文/章节/段落），保留元数据。
> 检索用混合方案——向量检索擅长语义，BM25 擅长术语，两者 RRF 融合后再 BGE-reranker 精排。"

### Agent

> "不是简单的一问一答。Agent 用 ReAct 模式，能思考、拆解任务、调用工具、观察结果、迭代。
> 比如用户说'帮我写综述'，Agent 会：先检索相关文献 → 逐章生成 → 传递前文摘要保持连贯 → 导出完整文档。"

## 5. 用户流程演示（核心环节）

### Phase 1: 创建项目

```
操作：上传 5-10 篇 PDF
系统行为：
  - 自动解析、按章节切片
  - 提取图表，生成描述
  - 向量化存入 Chroma
  - 提示"已索引 X 篇论文，可开始结构化总结"
```

### Phase 2: 结构化总结

```
操作：定义字段 [研究方法, 数据集, 核心指标, 创新点]
系统行为：
  - Agent 批量提取每篇论文的字段
  - 生成对比表格
  - 主动推荐"是否生成指标对比柱状图？"
  - 生成可视化图表
```

### Phase 3: 框架讨论

```
操作：用户说"帮我想想怎么写"
系统行为：
  - 分析表格数据，发现核心洞察
  - 推荐顶刊标准框架
  - 用户修改框架
  - 系统记录偏好
```

### Phase 4: 分节生成

```
操作：用户说"按框架写完整文章"
系统行为：
  - 逐章生成，每章先检索再生成
  - 滑动窗口保持连贯
  - 每个观点附带引用
  - 生成完毕导出 Markdown
```

## 6. Bad Case 展示（风险意识）

> "任何 AI 产品都必须考虑 Bad Case。CiteWise 的三层防护："

| 场景 | 防护 |
|------|------|
| 幻觉：编造引用 | Prompt 约束 + 后处理校验 |
| 提取遗漏：字段值为空 | 放宽检索重试 + 标记低置信度 |
| 连贯性：前后矛盾 | 滑动窗口摘要 + 一致性检查 |

> "关键是：AI 不确定的时候诚实说'需要更多信息'，不瞎编。"

## 7. 评估体系

> "没有评估就没有优化。CiteWise 对每个环节都有量化指标："

| 环节 | 指标 | 目标 |
|------|------|------|
| 检索 | 召回率/精确率/MRR | 95% / 85% / 0.8 |
| 提取 | 准确率/完整率 | 85% / 90% |
| 引用 | 准确率/幻觉率 | 90% / <5% |
| 生成 | 人工评分（1-5） | ≥4 |

## 8. Q&A 预设问题

### Q: 和 ChatGPT 写论文有什么区别？
> "三个核心区别：1) 强制溯源——每个观点必须引用知识库文献，不会幻觉；2) 可定制——字段、框架、风格都可定制；3) 记忆——跨项目复用偏好和模板。"

### Q: 为什么不用 RAG Agent 框架（LangGraph/LlamaIndex）？
> "MVP 阶段自己写更灵活，更好控制每个环节。生产环境可以考虑框架，但核心设计（切片策略、混合检索、溯源机制）是通用的。"

### Q: 幻觉问题能完全解决吗？
> "不能 100% 解决，但三层防护可以降到可接受水平。关键思路是：预防 + 检测 + 兜底。并且让用户能验证每一个引用的来源。"

### Q: 成本怎么控制？
> "国产模型（Qwen/DeepSeek）成本远低于 GPT-4。BGE embedding 和 reranker 可本地部署。MVP 预估单用户月成本 ≤ $1。"

### Q: 五大能力中哪个最难？
> "记忆工程。因为涉及隐式学习用户偏好、跨项目复用、状态管理，需要大量 bad case 才能调好。但也是差异化最大的——现有工具几乎没有跨会话记忆。"
