# CiteWise V3 — 产品需求文档 (PRD)

> **版本**: V3.1
> **更新日期**: 2026-04-13
> **产品定位**: AI 驱动的多 Agent 学术研究助手
> **技术栈**: FastAPI + LangGraph + 智谱 GLM + ChromaDB + Tailwind CSS

---

## 1. 产品概述

### 1.1 产品愿景

CiteWise 致力于成为研究人员的 **AI 学术副驾驶**，通过多 Agent 协同架构，将文献检索、知识管理、论文写作和数据分析整合为一体化工作流，让研究者专注于思考而非工具操作。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| **效率提升** | 多 Agent 并行协作，自动完成文献检索→知识抽取→论文生成全链路 |
| **质量保障** | RAG 检索 + 来源标注 + 引用验证，减少 AI 幻觉，确保学术严谨性 |
| **透明可控** | 实时 Agent 时间线可视化，用户可追踪每一步推理过程 |
| **灵活扩展** | 支持 6+ LLM 供应商切换，自定义 Agent 和 Skill 系统 |

### 1.3 核心差异化

1. **LangGraph Supervisor 多 Agent 编排** — 声明式状态图，非简单链式调用
2. **实时 Agent Timeline** — 可视化展示多 Agent 协作全过程
3. **三源标注系统** — [KB] 知识库 / [WEB] 联网 / [AI] 推理，来源透明
4. **混合检索引擎** — BM25 + 向量 + RRF 融合 + 重排序
5. **内置评估体系** — 5 大核心指标 + 自动优化建议

## 2. 目标用户

### 2.1 用户画像

| 角色 | 场景 | 核心需求 |
|------|------|----------|
| **博士研究生** | 文献综述、开题报告、论文写作 | 快速理解大量文献，生成结构化综述 |
| **科研工作者** | 课题调研、论文撰写、数据分析 | 跨文献对比分析，辅助论文各章节撰写 |
| **研究助理** | 文献管理、知识库维护、资料整理 | 批量处理文献，结构化提取关键信息 |

### 2.2 典型使用场景

**场景 A：文献综述撰写**
> 研究生小李需要撰写"电动汽车充电基础设施"的文献综述。上传 20 篇 PDF 后，在协同中心输入"总结当前文献中 Transformer 的应用趋势"，系统自动调度 Researcher 检索知识库、Responder 生成带引用标注的回答。

**场景 B：跨文献结构化对比**
> 王教授需要对比 10 篇论文的研究方法、核心算法和数据集。使用"提取矩阵"功能，自定义提取维度，一键生成对比表格并导出 Excel。

**场景 C：章节渐进式写作**
> 陈同学在"章节草稿"中创建"文献综述"章节，使用"智能续写"和子对话逐步完善内容，每次修改都基于 RAG 检索的文献依据。

## 4. 用户故事

### US-01: 多 Agent 协同问答
> **作为** 一名博士生，
> **我想要** 用自然语言提问并获得带引用标注的回答，
> **以便** 快速了解研究领域的最新进展并获取可靠文献来源。

**验收标准**：
- 输入问题后 3s 内开始流式输出
- Agent 时间线实时显示处理步骤
- 回答中包含 [作者, 年份] 格式的引用标注
- KB/WEB/AI 来源以不同颜色区分

### US-02: 文献上传与检索
> **作为** 一名研究者，
> **我想要** 批量上传 PDF 文件并自动建立知识库，
> **以便** 后续通过自然语言检索相关文献片段。

**验收标准**：
- 支持 PDF/DOC/DOCX/MD/TXT 格式
- 上传后自动解析、切片、向量化
- 上传进度实时展示
- 检索结果包含文献标题、作者、年份

### US-03: 章节智能生成
> **作为** 一名论文作者，
> **我想要** 基于 RAG 检索结果自动生成论文章节，
> **以便** 快速构建论文初稿。

**验收标准**：
- 支持引言/文献综述/方法论/研究结果/讨论/结论等预设章节
- 生成内容基于知识库检索，包含引用
- 支持通过子对话逐步修改和完善
- 生成后自动保存并可在 AgentEval 中评估

### US-04: 跨文献结构化对比
> **作为** 一名研究者，
> **我想要** 自定义提取维度并对多篇文献进行并行提取，
> **以便** 快速对比不同论文的研究方法和结论。

**验收标准**：
- 用户可自定义任意提取维度
- 提取结果以学术风格表格展示
- 支持导出为 CSV/Excel

### US-05: 模型灵活切换
> **作为** 一名用户，
> **我想要** 在不同 LLM 供应商之间切换，
> **以便** 根据任务需求和成本选择最合适的模型。

**验收标准**：
- 支持至少 6 个供应商（智谱/DeepSeek/OpenAI/Moonshot/千问/自定义）
- API Key 验证后才保存
- 切换模型后聊天立即可用新模型
- 模型选择器样式与整体 UI 一致

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标值 |
|------|--------|
| 首次 Token 延迟 (TTFT) | < 3s |
| 流式输出速率 | > 20 tokens/s |
| 文献上传解析 | < 30s/篇 |
| RAG 检索延迟 | < 2s |
| 页面首次加载 | < 2s (FCP) |

### 5.2 安全

| 要求 | 实现方式 |
|------|----------|
| API Key 安全 | 不存储明文到 URL/日志，前端 localStorage 加密存储 |
| 输入校验 | 后端 Pydantic schema 验证，消息长度限制 2000 字符 |
| 速率限制 | 30 请求/分钟/IP |
| 安全头 | X-Content-Type-Options / X-Frame-Options / Referrer-Policy |
| CORS 白名单 | 仅允许已知前端域名 |
| 用户数据隔离 | JWT Token + user_id 过滤 |

### 5.3 可用性

| 要求 | 标准 |
|------|------|
| 浏览器兼容 | Chrome 90+ / Firefox 90+ / Safari 15+ / Edge 90+ |
| 响应式布局 | 桌面端优先，支持 1280px+ 分辨率 |
| 国际化 | 界面中文，支持中英文混合内容处理 |
| 错误提示 | Toast 通知，不使用 alert/prompt |
| 无障碍 | 语义化 HTML，ARIA 标签，键盘导航 |

### 5.4 可靠性

| 要求 | 标准 |
|------|------|
| SSE 容错 | CRLF/LF 兼容解析，连接中断自动提示 |
| LLM 降级 | API 调用失败时返回友好错误信息 |
| RAG 容错 | 检索失败时退回纯 LLM 生成 |
| 数据持久化 | SQLite 持久存储 + ChromaDB 向量持久化 |

## 6. 数据指标与评估

### 6.1 核心 KPI

| 指标 | 计算方式 | 目标 |
|------|----------|------|
| **任务成功率** | 成功任务数 / 总任务数 | ≥ 85% |
| **引用准确率** | 已验证引用数 / 总引用数 | ≥ 80% |
| **幻觉率** | 未验证陈述数 / 总陈述数 | ≤ 10% |
| **平均响应时间** | 所有任务的平均完成时间 | ≤ 10s |
| **用户满意度** | 用户评分均值 | ≥ 4.0/5.0 |

### 6.2 自动优化规则

| 条件 | 建议 |
|------|------|
| 成功率 < 85% | 优化路由逻辑和错误处理 |
| 幻觉率 > 10% | 增强 RAG 检索质量，引入 CoVe 验证 |
| 响应时间 > 10s | 优化 LLM 调用链，减少 RAG top_k |
| 引用准确率 < 80% | 改进检索器重排序策略 |

### 6.3 评估方法

1. **自动评估**：每次聊天结束后自动记录指标到 eval.db
2. **人工评估**：用户可在 AgentEval 面板手动评分
3. **趋势分析**：7/14/30 天维度查看指标变化趋势
